# Estratégia de testes

## Tipos de testes realizados

Não foi localizada documentação da aplicação por isso os testes foram iniciados com testes exploratório para entendimento do fluxo e identificação de cenários.

- Teste exploratório
- Teste funcional
- Teste manual
- Validação de campos
- Testes positivos e negativos

## Técnicas de testes
Para otimizar a quantidade de testes a serem realizados,  utilizei  técnica caixa preta, que não consideram o código fonte.


- Análise de valor limite -  aplicada nos campos de nome e senha, com o objetivo de verificar o comportamento do sistema em relação aos limites mínimos e máximos de caracteres permitidos em cada campo.


## Critérios de aceite

 Os testes foram considerados satisfatórios quando:

- O sistema apresentou o comportamento esperado
- As validações funcionaram corretamente
- As mensagens de erro foram exibidas adequadamente
- O cadastro foi realizado com sucesso nos cenários válidos.
